# Add welcome modal with browser instructions

Show a welcome modal on first load (before the chat is visible) explaining:
1. They should accept/keep this WiFi network connected (devices may warn about no internet).
2. How to open the full chat in their browser if the captive portal mini-browser is limiting (show the URL 192.168.4.1).
3. Include a button/link that attempts to open 192.168.4.1 in the system browser.

Style: dark theme matching existing UI. Dismiss with a 'Got it' / 'Start chatting' button. Store dismissal in sessionStorage so it doesn't reappear on the same session but does appear on reconnect/new session.

Implementation in src/index_html.h — a simple overlay div, no external deps.

## Acceptance Criteria

First visit shows a modal explaining WiFi and browser instructions. Dismissing it reveals the chat. Modal does not reappear in the same session.
